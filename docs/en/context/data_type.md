# Data Type

When creating aclTensor through **aclCreateTensor** interface, for full data types supported, please refer to [acl API (C)](https://hiascend.com/document/redirect/CannCommunityCppApi) "Data Type and Its Operation Interface > aclDataType".
In two-stage interface parameter description, for convenience of description, supported data types will use the following abbreviated forms.

**Table 1** Data Type Abbreviation Table

| Original Data Type | Abbreviated Form (Case Insensitive) |
| :---------------: | :----------------------: |
|     ACL_FLOAT     |      FLOAT or FLOAT32      |
|    ACL_FLOAT16    |         FLOAT16          |
|     ACL_INT8      |           INT8           |
|     ACL_INT32     |          INT32           |
|     ACL_UINT8     |          UINT8           |
|     ACL_INT16     |          INT16           |
|    ACL_UINT16     |          UINT16          |
|    ACL_UINT32     |          UINT32          |
|     ACL_INT64     |          INT64           |
|    ACL_UINT64     |          UINT64          |
|    ACL_DOUBLE     |     DOUBLE or FLOAT64      |
|     ACL_BOOL      |           BOOL           |
|    ACL_STRING     |          STRING          |
|   ACL_COMPLEX64   |        COMPLEX64         |
|  ACL_COMPLEX128   |        COMPLEX128        |
|     ACL_BF16      |      BF16 or BFLOAT16      |
|     ACL_INT4      |           INT4           |
|     ACL_UINT1     |          UINT1           |
|   ACL_COMPLEX32   |        COMPLEX32         |
